项目一：Asking Bristol 公益资源匹配中台（B端/后台产品方向） 项目定位： 基于低代码架构的非盈利组织（NGO）资源供需匹配 SaaS 系统 在 Asking Bristol 项目中，我作为核心产品负责人，主导了从 0 到 1 的需求挖掘、产品定义与系统落地工作。该项目的核心背景是布里斯托当地的第三部门（Third Sector）面临严重的“资源错配”问题——虽然有大量志愿者和捐赠意愿，但NGO组织缺乏有效的渠道来获取其真正急需的专业技能（如法律咨询、IT支持）而非单纯的资金援助。针对这一宏观痛点，我首先通过双钻模型（Double Diamond）对布里斯托地区的 10+ 家核心 NGO 进行了半结构化深访与 User Journey Map 梳理 。调研数据揭示了一个反直觉的洞察：NGO 并不缺乏短期人力，最痛的点在于“需求描述非标准化”导致的沟通成本极高，以及缺乏一个闭环系统来反馈资助后的成果，导致捐赠者信任度流失。 基于此洞察，在产品方案设计阶段，我面临一个关键的权衡决策。团队初期曾设想开发原生 App，但考虑到 NGO 普遍缺乏技术维护资金且用户通过 PC 端办公的场景更重，我果断叫停了高成本的 App 开发计划，转而采用“轻量化 MVP”策略。我规划了一套基于 WordPress 前端嵌入 + Firebase 后端数据库的 Web 解决方案 。在功能设计上，我并未简单地堆砌表单，而是对“供需数据”进行了结构化重构。我定义了包含“技能类型（Skill Tags）”、“紧急程度（Urgency Level）”以及“服务时长”等关键字段的元数据标准，这使得系统能够实现自动化的供需筛选，替代了以往依靠邮件和人工 Excel 匹配的低效流程。为了解决信任痛点，我特别设计了“全链路反馈机制”与“数据可视化看板”，让捐赠者能直观看到资源的流向与产出 。 与AB创始人最后一次会议交流中，显著验证了产品策略的正确性。通过引入共享文档权限管理与自动化分类逻辑，我们将单次资源匹配的人力成本降低了约 40%-50%（基于流程节点测算），解决了长期困扰 Asking Bristol 的运维难题。更重要的是，这套低代码系统在无专职技术团队维护的情况下，依然保持了高可用性，实现了项目的可持续运营。这段经历极大地锻炼了我在资源受限环境下进行产品架构设计的能力，以及透过表象挖掘 B 端用户真实业务诉求的洞察力。

项目二：SmartWear 智能衣橱与循环经济平台（C端/AI产品方向） 项目定位： AI 驱动的数字化衣橱管理与闲置流转 App SmartWear 是我针对时尚行业“过度消费与闲置浪费”这一矛盾，结合 Gen Z 人群消费习惯孵化的创新型 C 端产品。项目的核心切入点是解决用户“衣橱塞满却无衣可穿”的决策瘫痪，以及闲置衣物“想卖却嫌麻烦”的流转摩擦。通过对 3500 亿美元规模的二手市场进行 PESTLE 分析及竞品调研（如 Depop, Whering），我发现现有竞品要么只做工具（衣橱管理），要么只做交易（二手电商），导致用户体验割裂 。因此，我将 SmartWear 定义为“Manage-to-Sell”的一站式闭环平台。 在核心功能设计上，我将“降低用户录入成本”作为 MVP 阶段的北极星指标。传统的衣橱 App 需要用户手动填写大量信息，流失率极高。为此，我规划了基于 AI 图像识别技术的录入模块：用户只需拍照，系统即可自动通过 CV 算法解析并填充面料、颜色、品牌、剪裁等 10+ 个 SKU 属性 。这不仅极大提升了用户体验，更为后续的“一键转售”打下了标准化的数据基础。为了打通商业闭环，我设计了跨平台分发系统（API Integration），允许用户将闲置衣物一键同步至 eBay、Depop 等主流二手平台，并利用 AI 算法根据历史成交数据提供“智能定价建议”，解决了非专业卖家不懂定价的痛点 。 在商业化与增长策略方面，我摒弃了单一的广告变现，设计了 Freemium（免费+增值）的混合盈利模型。基础功能免费以做大 DAU（日活），而将“AI 深度搭配建议”、“多平台批量管理”及“数据分析报表”作为 Tier 1-5 的付费点，精准收割高净值用户与职业二手卖家（Pro-Resellers） 。同时，我规划了通过微型网红（Micro-influencers）进行冷启动的 GTM 策略，利用“晒出你的 AI 穿搭”等社交裂变玩法获取种子用户。该项目不仅展现了我对 AI 技术落地场景的把控能力，更体现了从用户体验到商业变现的全链路产品思维。

项目三：Circlo 纺织废料 B2B2C 匹配平台（平台/供应链方向） 项目定位： 连接品牌库存与独立设计师的供应链协同 SaaS 平台 Circlo 项目诞生的背景是时尚产业每年产生 9200 万吨废料，而大量中小品牌（Emerging Brands）受限于供应链能力，只能将死库存（Deadstock）焚烧或填埋 。作为一个典型的双边市场问题，我识别出供给端（品牌）有处理库存的合规压力，而需求端（独立设计师/再造者）缺乏稳定且低成本的面料来源。基于此，我设计了 Circlo 这一 B2B2C 平台，旨在重构废旧纺织品的供应链路。 在平台机制设计上，核心挑战在于如何实现非标品的“标准化匹配”。品牌方的库存千奇百怪，设计师的需求也各不相同。我主导设计了 B 端的库存管理后台，强制推行结构化的 SKU 录入标准（成分、克重、瑕疵程度等），并建立了一套基于标签（Tag-based）的双向匹配算法 。系统能够根据设计师的“擅长风格”与品牌方的“面料属性”进行自动撮合，将传统的线下寻源效率提升了数倍。为了解决 B 端用户的活跃度问题，我深入挖掘了品牌方的 ESG（环境、社会和治理）诉求，在产品中创新性地植入了“影响力仪表盘（Impact Dashboard）”。该功能将每一次交易和改造量化为“碳减排数据”和“废料转移量”，为品牌方提供了可用于 CSR 报告的合规背书，从而极大地提升了供给侧的留存率 。 此外，为了验证 C 端市场的接受度，我还设计了“全链路溯源（Traceability）”的前端体验。消费者在购买再造衣物时，可以查看到面料的“前世今生”——它来自哪个品牌的哪件库存，经过了哪位设计师的改造。这种 Storytelling（故事化）的产品设计，成功将“旧衣”转化为具备环保溢价的“潮流单品” 。Circlo 项目充分展示了我处理复杂双边市场关系、设计供应链业务流程以及利用数据价值赋能 B 端客户的高阶产品能力。
